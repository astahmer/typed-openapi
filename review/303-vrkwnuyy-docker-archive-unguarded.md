## Comment

Per-runtime tstyche suites under `tests/tstyche/{runtimes,effect-client}/` are generated by `gen-tstyche-suites.ts` and
guarded by `gen-tstyche-suites.test.ts`. The docker archive regression suite is a **manual** file importing
`tmp/tstyche/docker-effect/client.ts` (fixture from `gen-tstyche-fixtures.ts`).

If `ApiCallParams` / error-channel emission changes, the generated suites refresh on `pnpm gen:tstyche-suites`, but
`docker-archive.types.tstyche.ts` can silently drift. Either emit it from the generator (docker fixture already exists
in `gen-tstyche-fixtures.ts`) or add a minimal guard test that asserts key strings
(`TypedStatusError | HttpClientError`, archive path/query shapes).

## Evidence

- `gen-tstyche-fixtures.ts` writes `tmp/tstyche/docker-effect/client.ts` with
  `endpointPatterns: ["/containers/{id}/archive"]`.
- `docker-archive.types.tstyche.ts` is not referenced in `gen-tstyche-suites.ts` or `gen-tstyche-suites.test.ts`.
- `qwtlwsnm` added sync tests for zod promise/effect suites only, not docker-archive.
